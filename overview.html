The LDAP Chai API, easy to use LDAP for Java developers. See also the LDAP Chai Project Page.

LDAP Chai is a wrapper library that makes using ldap services as simple and easy as it can be. LDAP Chai has methods for commonly used functions, and relieves developers of the need to write code that has been written many times before.

Several useful features make LDAP Chai an ideal choice for building ldap applications:

  • ldap fail-over - allows your application to easily take advantage of mirrored/replicated ldap directories that are common in enterprise environemnts. see {@link com.novell.ldapchai.provider.ChaiSetting#FAILOVER_ENABLE}.
  • idle connection watchdog - closes and re-opens ldap connections (sockets) automatically based on activity, keeping the connection count lower than it would be otherwise {@link com.novell.ldapchai.provider.ChaiSetting#WATCHDOG_ENABLE}.
  • pluggable provider interface - wraps commonly used low level LDAP APIs such as JNDI ldap api or JLDAP api for maximum compatibility. {@link com.novell.ldapchai.provider.ChaiSetting#PROVIDER_IMPLEMENTATION}.

LDAP Chai also has a rich library of password mangement, policy and forgotten password methods.

{@link com.novell.ldapchai.provider.ChaiProvider} is the foundation interface for LDAP Chai, and contains the basic ldap accessor methods. Direct use of {@code ChaiProvider} is discouraged, instead use the wrapper class {@link com.novell.ldapchai.ChaiEntry} and its subclasses wherever practical. {@code ChaiEntry} instances wrap an LDAP DN, and relieve the caller of the necessity of working directly with String DNs.

{@code ChaiProvider} instances can be obtained using {@link com.novell.ldapchai.provider.ChaiProviderFactory}. Similarly, {@code ChaiEntry} instances can be obtained using {@link com.novell.ldapchai.ChaiFactory}.
